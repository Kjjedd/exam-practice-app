# feature/exam-mode Plan

## Goal

일반 학습 모드와 구분되는 시험형 연속 풀이 경험을 제공하되, 기존 세션/결과 구조를 최대한 재사용하면서 시험 모드에 맞는 흐름과 UI 규칙을 구현한다.

## Branch Intent

이 브랜치는 "학습 중심 문제풀이"를 "시험처럼 연속해서 푸는 세션"으로 확장하는 단계다.

- [x] 시험 모드 진입 시 일반 모드와 다른 세션 경험을 제공한다.
- [x] 시험 모드에서는 현재 단계에 맞는 UI/UX 차이를 분명히 드러낸다.
- [x] 시험 모드 결과가 기존 결과 요약 구조와 충돌하지 않도록 유지한다.
- [x] 다음 단계인 즐겨찾기 기능과 섞이지 않도록 시험 모드 책임을 분명히 유지한다.

이번 단계의 핵심은 "시험처럼 연속적으로 푸는 경험"이지, 통계 저장이나 시험 기록 보관이 아니다.

## Scope

- 시험 모드 진입
- 시험 모드 전용 세션 규칙
- 시험형 문제 화면 문구/레이아웃 조정
- 시험 모드의 문제 전환 UX 조정
- 시험 모드 결과 화면 연결
- 시험 모드 세션 정보 저장

## Detailed Scope

이번 단계에서 다뤄야 하는 핵심은 아래와 같다.

- `mode=exam` 진입 시 일반 모드와 다른 컨텍스트를 분명히 보여주기
- 기존 문제풀이 구조를 재사용하되, 시험 모드에서는 학습형 설명 문구를 덜고 집중형 흐름을 주기
- 시험 모드가 랜덤/복습과 충돌하지 않도록 세션 규칙을 분리하기
- 결과 화면에서 시험 모드 세션이라는 사실을 자연스럽게 표시하기
- 시험 모드에서도 결과 집계와 오답 식별 구조는 그대로 유지하기

이번 브랜치에서는 타이머, 제한 시간, 자동 제출 같은 강한 시험 제약까지는 포함하지 않는다. 지금 단계의 시험 모드는 "연속 풀이 중심 UX"를 먼저 확보하는 것이 목적이다.

## State Principles

시험 모드 상태는 아래 원칙을 따라야 한다.

- 시험 모드 여부는 `QuizMode`로 식별 가능해야 한다.
- 시험 모드도 기존 세션 구조와 호환되어야 한다.
- 결과 집계 규칙은 일반 모드와 동일해야 한다.
- 시험 모드 특화 차이는 주로 UX와 화면 정보 밀도에서 만들어야 한다.
- 복습/랜덤과 동시에 활성화되는 복합 모드는 이번 단계에서 허용하지 않는다.

## Proposed Data Shape

이번 단계에서는 최소한 아래 수준의 구조가 필요하다.

- `mode: "exam"`
- 기존 `QuizSession`
- 필요 시 시험 모드용 UI 플래그 또는 유도 값:
  - `isExamMode: boolean`
  - `modeLabel`
  - `modeDescription`

핵심은 기존 세션 타입을 깨지 않고 시험 모드 차이를 얹는 것이다.

## Exam-mode UX Rules

시험 모드 UX는 최소한 아래 규칙을 만족해야 한다.

- 사용자는 현재 시험 모드임을 화면에서 즉시 인식할 수 있어야 한다.
- 문제 간 이동은 일반 모드보다 덜 학습 보조적으로 보여야 한다.
- 화면은 더 집중감 있게 보여야 하지만, 기존 풀이 구조를 크게 깨면 안 된다.
- 결과 화면에서는 시험 모드로 풀었다는 점을 다시 확인할 수 있어야 한다.

### Navigation Constraint Revision

이후 리비전에서는 학습 모드에 자유 이동이 추가되더라도, 시험 모드는 현재의 연속 응시 제약을 유지해야 한다.

- 시험 모드에서는 이전 문항으로 돌아가는 CTA를 제공하면 안 된다.
- 시험 모드에서는 미제출 상태에서 다음 문항으로 건너뛰는 자유 이동도 허용하면 안 된다.
- 시험 모드에서 허용되는 이동은 현재 문항 제출 후 다음 문항으로 진행하는 흐름과 마지막 결과 보기 CTA뿐이다.
- 같은 `QuizNavigation` 컴포넌트를 재사용하더라도, 시험 모드일 때는 자유 이동 관련 UI와 동작이 명확히 비활성화되어야 한다.

## Interaction Design

이번 단계에서 기대하는 상호작용은 다음과 같다.

### 1. From Home

- 사용자는 홈에서 시험 모드를 선택한다.
- 시험 모드로 들어오면 헤더와 안내 문구가 일반 모드와 다르게 보인다.

### 2. During Quiz

- 사용자는 연속 풀이 중심의 화면을 본다.
- 시험 모드에서도 답 선택, 제출, 다음 문제 이동은 유지되지만, 문맥상 "연습"보다 "응시"에 가깝게 느껴져야 한다.

### 3. Session Complete

- 마지막 문제 이후 결과 화면으로 이동한다.
- 결과 화면은 시험 모드 세션임을 설명해 준다.

### 4. Non-goals in This Step

- 제한 시간
- 타이머
- 자동 채점 제출
- 시험 이력 저장

## UI Principles

- 시험 모드는 일반 학습 모드보다 더 집중감 있고 덜 설명적인 분위기가 적합하다.
- 하지만 현재 앱 구조 안에서는 과도한 UX 분리를 피하고, 재사용 가능한 컴포넌트 위에 차이를 얹는 편이 안전하다.
- 시험 모드 표시는 헤더/설명/CTA 문구 수준에서 분명해야 한다.
- 결과 화면은 모드 차이를 드러내되, 집계 방식은 동일하게 유지해야 한다.

## Data Flow Notes

현재 세션 구조는 `mode`, `questionIds`, `results`를 저장하고 있다.  
시험 모드에서는 아래 흐름이 중요하다.

- 시험 모드 진입
- 시험 모드 라벨과 설명으로 세션 시작
- 기존 문제풀이/결과 저장 구조 재사용
- 결과 화면에서 시험 모드 설명 반영

즉, 시험 모드의 핵심은 새로운 저장 구조보다 "시험형 UX 규칙을 기존 세션에 얹는 것"이다.

## Risks

- 시험 모드 차이를 과도하게 크게 만들면 기존 세션 재사용성이 떨어질 수 있다.
- 랜덤/복습과의 경계가 모호해지면 모드 분기 유지보수가 어려워질 수 있다.
- 시험 모드 설명만 바꾸고 실제 UX 차이가 없으면 기능 의미가 약해질 수 있다.
- 타이머 같은 비범위 요소를 섞기 시작하면 브랜치 책임이 커질 수 있다.

- 결과와 복습의 일관성을 깨지 않는 시험 모드가 가장 중요하다.

## Current Exam Template Revision

추가 수정에서는 시험 모드를 "일반 시험 느낌의 모드"에서 "실제 현행 시험 템플릿 선택형 모드"로 확장해야 한다.

- 시험 모드 진입 전 사용자가 실제 시험 템플릿을 선택할 수 있어야 한다.
- 최소한 현재 지원하는 AWS 현행 시험 목록을 템플릿으로 제공해야 한다.
- 각 시험 템플릿은 다음 정보를 가져야 한다.
  - 시험 코드
  - 시험명
  - 총 문항 수
  - 채점 방식 설명
  - 합격 기준 설명
- 시험 세션은 선택한 템플릿 기준으로 문제 수를 잘라서 랜덤 출제해야 한다.
- 결과 화면에서도 어떤 시험 템플릿으로 응시했는지와 연습용 판정 기준을 보여줄 수 있어야 한다.

공식 scaled scoring의 내부 배점 계산은 AWS가 공개하지 않으므로, 앱에서는 "공식 합격 점수/채점 방식 설명"과 "연습용 합격 판정 기준"을 분리해 다루는 편이 안전하다.

### Current Exam Template Deliverables

- [x] 시험 템플릿 데이터 정의
- [x] 시험 선택 화면 또는 선택 패널
- [x] 시험 템플릿 기반 문항 수 제한 및 랜덤 출제
- [x] 시험 세션에 선택 템플릿 정보 저장
- [x] 결과 화면의 시험 템플릿 정보 및 연습용 합격/불합격 표시

### Current Exam Template Validation

- [x] 예를 들어 SAA를 선택하면 SAA 템플릿의 문항 수 기준으로 세션이 만들어져야 한다.
- [x] 결과 화면에 시험 코드/시험명/문항 수/합격 기준이 반영되어야 한다.
- [x] 공식 비공개 배점은 추정해서 표기하지 않아야 한다.

## Deferred Scoring Revision

추가 수정에서는 시험 모드의 채점 시점을 일반 학습 모드와 분리해야 한다.

- 시험 모드에서는 보기 클릭 시 답안만 저장되고 즉시 정답/오답을 노출하면 안 된다.
- 시험 모드에서는 문제 화면에서 정답 하이라이트, 오답 하이라이트, 해설 패널이 나타나면 안 된다.
- 시험 모드의 정답/오답과 합격/불합격 판단은 마지막 결과 화면에서만 확인 가능해야 한다.
- 시험 모드에서도 내부 결과 데이터는 계속 저장하되, UI 노출만 지연하는 방식이 안전하다.

- [x] 시험 모드 해설 패널 비노출

- [x] 시험 모드 결과는 마지막 결과 화면에서만 확인 가능해야 한다.
