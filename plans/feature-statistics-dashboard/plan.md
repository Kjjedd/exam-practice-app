# feature/statistics-dashboard Plan

## Goal

사용자가 현재 학습 상태를 한눈에 이해할 수 있도록, 저장된 문제 세트/최신 세션/즐겨찾기 정보를 기반으로 핵심 학습 통계를 보여주는 대시보드를 만든다.

## Branch Intent

이 브랜치는 지금까지 저장 계층에 쌓인 사용자 데이터를 "의미 있는 학습 지표"로 해석해 보여주는 단계다.

- [x] 대시보드는 최신 세션과 저장된 사용자 상태를 읽어 핵심 학습 요약을 계산한다.
- [x] 사용자는 정답률, 풀이 수, 오답 수, 즐겨찾기 수, 마지막 학습 시점 같은 정보를 한 화면에서 확인할 수 있어야 한다.
- [x] 저장 데이터가 없거나 불완전해도 대시보드 화면은 깨지지 않고 안전한 빈 상태를 보여야 한다.
- [x] 이후 저장 구조 리팩터링이나 버그 수정 없이도 현재 저장 계층 위에서 안정적으로 동작하는 통계 해석 기준을 마련한다.

이번 단계의 핵심은 "그래프를 화려하게 그리는 것"이 아니라, 현재 저장된 학습 데이터를 사용자가 이해하기 쉬운 요약 정보로 신뢰성 있게 표현하는 것이다.

## Scope

- `/dashboard` 화면 구현
- 통계 계산 로직 구현
- 최신 세션 기반 학습 요약 계산
- 즐겨찾기 수와 저장된 문제 세트 상태 요약
- 데이터가 없는 경우의 빈 상태 처리
- 저장 계층과 UI 사이의 해석 경계 정리

## Detailed Scope

이번 단계에서 다뤄야 하는 핵심은 아래와 같다.

- 최신 풀이 세션에서 총 풀이 수, 정답 수, 오답 수, 정답률을 계산하기
- 즐겨찾기 저장소에서 현재 즐겨찾기 수를 반영하기
- 저장된 문제 세트 개수와 활성 문제 세트 존재 여부를 대시보드 문맥에서 보여줄지 결정하기
- 마지막 학습 시점을 최신 세션의 `completedAt` 또는 `startedAt` 기준으로 안전하게 계산하기
- 저장 데이터가 없을 때도 "아직 학습 기록이 없음"을 이해 가능한 화면으로 보여주기
- 결과 화면용 요약 로직과 대시보드 요약 로직이 중복되더라도 역할을 분리할지, 공통 유틸로 묶을지 기준을 세우기

이번 브랜치에서는 아직 장기 누적 통계 히스토리나 차트 시계열을 만들지 않는다. 현재 저장 구조상 신뢰할 수 있는 것은 "최신 세션 + 현재 저장 상태"이므로, 이 범위 안에서 정직한 통계를 제공한다.

## Statistics Principles

대시보드 통계는 아래 원칙을 따라야 한다.

- 통계는 실제 저장된 데이터만 바탕으로 계산한다.
- 없는 데이터를 추정해서 그럴듯하게 보이게 만들지 않는다.
- 최신 세션 중심 통계와 저장 상태 중심 통계를 구분한다.
- 정답률은 `correctCount / totalSolved`를 기준으로 계산한다.
- 문제 세트/즐겨찾기/세션 데이터가 서로 맞지 않더라도 화면은 깨지지 않아야 한다.
- 대시보드는 저장 계층의 내부 구현보다 "사용자가 이해하는 요약"을 우선한다.

## Current Data Context

현재 프로젝트에서 대시보드가 읽을 수 있는 저장 데이터는 아래와 같다.

- `QuestionBank`
  - 활성 문제 세트 ID
  - 저장된 문제 세트 목록
- `Latest Quiz Session`
  - 가장 최근에 완료된 또는 저장된 세션
  - 문제 수, 결과, 시작/종료 시각
- `favoriteQuestionIds`
  - 사용자가 즐겨찾기한 문제 ID 목록

즉, 현재 대시보드는 "장기 누적 DB"가 아니라 "현재 저장된 학습 상태"를 읽어 요약하는 구조다.

## Proposed Dashboard Data Shape

이번 단계에서는 최소한 아래 수준의 데이터가 필요하다.

- `StudyStats`
  - `totalSolved`
  - `correctCount`
  - `wrongCount`
  - `correctRate`
  - `favoriteCount`
  - `lastStudiedAt`
- 필요 시 추가 대시보드 전용 유도 값
  - `questionSetCount`
  - `hasActiveQuestionSet`
  - `latestModeLabel`
  - `latestQuestionSetTitle`

핵심은 `StudyStats`는 숫자 중심 요약을 담당하고, 화면 설명에 필요한 문맥 정보는 별도 유도 값으로 계산하는 것이다.

## Aggregation Rules

통계 계산 규칙은 최소한 아래를 만족해야 한다.

- 최신 세션이 완료된 세션일 때만 정답/오답/정답률 집계를 신뢰한다.
- 완료되지 않은 세션은 대시보드의 메인 성과 지표에 포함하지 않는다.
- `totalSolved`는 현재 구조상 최신 완료 세션의 문제 수를 기준으로 계산한다.
- `correctCount`는 `QuestionResult.isCorrect === true` 개수다.
- `wrongCount`는 `totalSolved - correctCount` 또는 오답 결과 개수와 일치해야 한다.
- `correctRate`는 `totalSolved === 0`이면 `0`, 아니면 반올림된 백분율이다.
- `favoriteCount`는 저장된 즐겨찾기 ID 개수다.
- `lastStudiedAt`는 완료 세션이 있으면 `completedAt`, 없으면 `startedAt`, 둘 다 없으면 `null`이다.

## Restore and Empty-state Rules

대시보드는 아래 상황을 안전하게 처리해야 한다.

- 저장된 세션이 없으면 "아직 학습 기록이 없습니다" 빈 상태를 보여야 한다.
- 활성 문제 세트가 없더라도 저장된 즐겨찾기나 세션 요약이 있으면 가능한 범위만 보여줘야 한다.
- 최신 세션이 깨졌거나 불완전하면 숫자 통계는 기본값으로 내리고, 안내 문구로 보완해야 한다.
- 즐겨찾기가 없으면 `favoriteCount = 0`으로 자연스럽게 표시해야 한다.

## Interaction Design

이번 단계에서 기대하는 상호작용은 다음과 같다.

### 1. Overview

- 사용자는 대시보드에 들어오자마자 현재 학습 요약을 본다.
- 가장 중요한 수치가 상단에서 바로 읽혀야 한다.

### 2. Context

- 사용자는 이 통계가 어떤 세션과 어떤 저장 상태를 기준으로 계산된 것인지 이해할 수 있어야 한다.
- 최신 학습 시점과 활성 문제 세트 관련 문맥이 함께 보이면 좋다.

### 3. Empty State

- 아직 풀이 세션이 없으면 홈 또는 퀴즈로 이동할 수 있는 자연스러운 CTA가 필요하다.

### 4. Non-goals in This Step

- 시계열 그래프
- 여러 세션 누적 히스토리
- 날짜별 분석
- 카테고리별 성과 비교

## UI Principles

- 수치는 한눈에 읽히는 카드 구조가 적합하다.
- 빈 상태와 실제 통계 상태는 시각적으로 명확히 구분돼야 한다.
- 설명 문구는 숫자의 의미를 보조하되, 수치 자체를 가리지 않아야 한다.
- 홈 화면과 결과 화면 사이의 중간 성격이 아니라, "내 학습 상태 요약"이라는 독립된 성격이 느껴져야 한다.

## Data Flow Notes

현재 구조를 기준으로 대시보드 데이터 흐름은 아래처럼 해석할 수 있다.

1. 저장 계층에서 현재 질문 세트 상태를 읽는다.
2. 최신 세션 저장소에서 마지막 세션을 읽는다.
3. 즐겨찾기 저장소에서 현재 즐겨찾기 개수를 읽는다.
4. 대시보드용 통계 유틸이 위 데이터를 숫자 요약으로 변환한다.
5. `/dashboard` 화면은 이 요약을 카드와 보조 설명으로 렌더링한다.

즉, 대시보드의 핵심은 저장소에서 직접 렌더링하는 것이 아니라, 저장 데이터를 "대시보드 문맥"으로 번역하는 데 있다.

## Risks

- 최신 세션만 가지고 누적 통계처럼 보이게 만들면 사용자 기대와 어긋날 수 있다.
- 결과 화면 요약과 대시보드 통계 계산식이 다르면 혼란을 줄 수 있다.
- 활성 문제 세트와 최신 세션 관계가 어긋난 상황을 제대로 처리하지 않으면 대시보드가 비정상적으로 비어 보일 수 있다.
- 저장 구조 이상의 것을 보여주려 하면 가짜 통계를 만들 위험이 있다.

## Notes

- 현재 저장 구조 기준으로는 "장기 누적 학습 분석"보다 "현재 저장 상태 요약"에 더 가깝다.
- 따라서 문구와 숫자 해석 모두 과장 없이 정직해야 한다.
- 이후 여러 세션 히스토리가 저장되기 전까지는, 최신 세션 중심 대시보드가 가장 안전하다.
