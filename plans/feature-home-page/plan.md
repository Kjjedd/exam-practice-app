# feature/home-page Plan

## Goal

사용자가 각 학습 흐름으로 진입할 수 있는 메인 화면과 최소 네비게이션 구조를 만든다.

## Branch Intent

이 브랜치는 "프로젝트 부트스트랩용 임시 홈 화면을 실제 제품의 시작 화면으로 교체한다"는 목적만 담당한다.

- [x] 사용자가 앱의 주요 흐름을 한눈에 이해할 수 있는 메인 화면을 만든다.
- [x] 일반/랜덤/시험 모드 진입점을 명확히 제공한다.
- [x] 복습/즐겨찾기/통계 화면으로 이동할 수 있는 링크를 배치한다.
- [x] 아직 구현되지 않은 하위 기능은 홈 화면에서 자연스럽게 placeholder 경로로 연결한다.

이 단계에서는 홈 화면과 진입 흐름만 다루며, 실제 문제풀이 로직은 구현하지 않는다.  
현재 브랜치에서는 이 원칙을 지킨 상태로 홈 화면과 진입 구조만 구현했다.

## Scope

- 홈 화면 레이아웃 구성
- 일반 모드 진입 UI
- 랜덤 모드 진입 UI
- 시험 모드 진입 UI
- 복습/즐겨찾기/통계 화면으로 가는 진입 링크 구성
- 각 진입 요소의 최소 라우팅 연결 또는 placeholder 경로 준비
- 반응형 홈 화면 구조 구성

## Detailed Scope

이번 단계에서 다뤄야 하는 핵심은 아래와 같다.

- 현재 부트스트랩 전용 홈 화면을 제품 소개형 홈 화면으로 대체
- 앱의 핵심 목적을 짧게 설명하는 상단 섹션 구성
- 주요 모드 진입 카드 또는 버튼 구성
- 보조 진입 링크(복습/즐겨찾기/통계) 배치
- 이후 브랜치에서 연결이 자연스럽게 이어질 수 있는 정보 구조 정리

이번 브랜치의 핵심은 "시작 화면이 앱의 사용 흐름을 설명하고, 다음 행동을 제안하는 것"이다.

## UX Principles

홈 화면은 아래 원칙을 따라야 한다.

- 사용자가 처음 봐도 무엇을 하는 앱인지 바로 이해할 수 있어야 한다.
- 주요 시작 행동은 보조 링크보다 더 강조되어야 한다.
- 일반 모드, 랜덤 모드, 시험 모드는 서로 구분되는 진입점이어야 한다.
- 복습, 즐겨찾기, 통계는 보조 기능이지만 홈에서 접근 가능해야 한다.
- 반응형 기준에서 모바일과 데스크톱 모두 자연스럽게 읽혀야 한다.

## Information Architecture

이번 단계에서 홈 화면에 포함되면 좋은 구조는 다음과 같다.

### 1. Hero Section

역할:

- 앱 소개
- 핵심 가치 전달
- 현재 사용자가 무엇을 할 수 있는지 요약

예상 내용:

- 앱 이름 또는 제목
- 시험 대비/문제풀이 앱이라는 설명
- 학습 흐름에 대한 짧은 소개 문장

### 2. Primary Actions

역할:

- 핵심 시작 흐름 제공

포함 대상:

- 일반 문제풀이 시작
- 랜덤 모드 시작
- 시험 모드 시작

고려사항:

- 세 가지 액션은 서로 시각적으로 구분되지만 같은 그룹으로 묶이는 편이 좋다.
- 이후 실제 라우팅이나 모드 전달 방식이 바뀌더라도 홈 화면 구조는 유지 가능해야 한다.

### 3. Secondary Links

역할:

- 보조 학습 기능 진입

포함 대상:

- 오답 복습
- 즐겨찾기 보기
- 통계 대시보드 보기

고려사항:

- 아직 하위 기능이 완성되지 않았더라도 경로 진입 자체는 준비할 수 있다.
- placeholder 상태라도 홈 화면에서 정보 구조가 드러나는 것이 중요하다.

### 4. Supportive Summary

역할:

- 앱이 제공할 학습 경험을 짧게 정리

예시:

- 즉시 채점
- 해설 확인
- 오답 복습
- 학습 기록 누적

이 구역은 선택 사항이지만, 홈 화면을 단순 버튼 목록보다 더 제품답게 보이게 만들어 준다.

## Out of Scope

- 실제 문제풀이 동작
- 문제 선택 상태 처리
- 채점/해설/결과 로직
- 저장/복원 로직
- 통계 계산 로직
- 즐겨찾기 토글 기능 구현
- 실제 오답 복습 기능 구현
- 모드별 세부 동작 구현
- 세션 상태 전달 로직 확정

- `app/page.tsx`
- `components/home/HomeHero.tsx`

- 필요 시 `components/home/HomeSection.tsx`

실제 파일 분리는 과도하게 하지 않되, 홈 화면이 커지면 `app/page.tsx` 하나에 모든 마크업을 몰아넣지 않는 편이 좋다.

## Routing Notes

현재 기준으로 홈 화면에서 연결 가능한 경로는 아래와 같다.

- 일반 문제풀이: `/quiz`
- 랜덤 모드: `/quiz` 기반 경로 또는 이후 확장 가능한 방식
- 시험 모드: `/quiz` 기반 경로 또는 이후 확장 가능한 방식
- 복습: `/review`
- 즐겨찾기: `/favorites`
- 통계: `/dashboard`

이번 단계에서는 실제 모드 동작보다 "사용자가 해당 흐름으로 이동할 수 있는 홈 화면 진입점"이 더 중요하다.

## Dependency Notes

이 단계는 아래 브랜치들과 연결된다.

- `feature/question-data`에서 준비된 실제 문제 데이터가 존재하므로, 홈 화면은 이제 제품 진입점 역할을 할 수 있다.
- `feature/question-ui`는 홈 화면에서 연결한 `/quiz` 흐름을 실제 문제 화면으로 발전시킨다.
- `feature/random-quiz-mode`, `feature/exam-mode`는 이번 홈 화면에서 만든 진입점을 나중에 실제 동작으로 채운다.
- `feature/wrong-answer-review`, `feature/favorites`, `feature/statistics-dashboard`는 이번에 노출된 보조 링크 흐름을 실제 기능으로 완성한다.

즉, 이번 홈 화면은 단지 현재를 위한 페이지가 아니라 이후 기능 브랜치들이 사용자 흐름을 이어받는 시작점이다.

- [x] 일반/랜덤/시험 모드 진입 요소가 명확히 구분된다.
- [x] 복습/즐겨찾기/통계 진입 링크가 화면에 노출된다.
- [x] 연결된 라우트 또는 placeholder 경로로 이동 가능하다.
- [x] 모바일/태블릿/데스크톱에서 반응형 레이아웃이 깨지지 않는다.
- [x] `npm run typecheck` 기준으로 새 오류를 만들지 않아야 한다.

아래 조건을 만족하면 `feature/home-page` 단계가 완료된 것으로 본다.

- [x] 부트스트랩용 임시 홈 화면이 실제 제품 진입 화면으로 교체된다.
- [x] 사용자가 홈에서 주요 학습 시작 행동을 바로 이해할 수 있다.
- [x] 일반/랜덤/시험 흐름이 분리된 진입 요소로 제공된다.
- [x] 복습/즐겨찾기/통계로 이동 가능한 링크가 노출된다.
- [x] 아직 미완성 기능도 홈 화면 정보 구조 안에서 자연스럽게 설명된다.
- [x] 레이아웃이 모바일과 데스크톱에서 모두 읽기 좋다.

## Risks

이번 단계에서 특히 주의해야 할 위험은 다음과 같다.

- 홈 화면이 설명만 많고 실제 행동 유도가 약하면 시작 화면 역할을 못 하게 된다.
- 반대로 버튼만 많고 제품 설명이 없으면 앱의 목적이 흐려질 수 있다.
- 아직 구현되지 않은 하위 기능을 과하게 완성된 것처럼 보이게 만들면 사용 흐름이 어색해질 수 있다.
- 홈 화면 안에서 모드별 세부 동작까지 미리 확정하면 이후 브랜치에서 수정 비용이 커질 수 있다.

## Notes

- 홈 화면은 현재 앱의 진입 허브 역할에 집중한다.
- 제품 소개와 행동 유도 사이의 균형이 중요하다.
- 현재 데이터 기반이 준비되어 있으므로, 홈 화면은 더 이상 단순 부트스트랩 안내 화면일 필요가 없다.
- 이번 브랜치에서는 "제품다운 첫 화면"을 만드는 것이 핵심이다.
