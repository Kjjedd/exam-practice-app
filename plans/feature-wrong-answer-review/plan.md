# feature/wrong-answer-review Plan

## Goal

방금 완료한 세션 결과를 기준으로 틀린 문제만 따로 식별하고, 사용자가 오답 목록을 확인한 뒤 그 문제들만 다시 풀 수 있는 복습 흐름을 만든다.

## Branch Intent

이 브랜치는 "결과를 읽는 것"에서 끝나지 않고 "틀린 문제를 다시 학습하는 것"으로 이어주는 단계다.

- [x] 결과 집계에서 오답 문제만 정확히 추출할 수 있는 기준을 만든다.
- [x] 결과 화면 또는 복습 화면에서 오답 목록을 확인할 수 있게 한다.
- [x] 오답 문제만 다시 푸는 전용 복습 세션을 시작할 수 있게 한다.
- [x] 다음 단계인 랜덤/시험 모드와 충돌하지 않도록 복습 흐름의 범위를 분명히 유지한다.

이번 단계의 핵심은 "오답만 다시 학습할 수 있게 만드는 것"이지, 전체 퀴즈 흐름을 다시 설계하는 것이 아니다.

## Scope

- 오답 목록 추출 로직
- 오답 전용 요약 또는 진입 UI
- 오답 복습 시작 CTA
- 오답 문제만 포함한 복습 세션 준비
- 복습 세션용 최소 상태 연결
- 오답이 없는 경우의 예외 상태 처리

## Detailed Scope

이번 단계에서 다뤄야 하는 핵심은 아래와 같다.

- 결과 화면에서 `isCorrect === false`인 문제를 식별할 수 있는 구조를 만든다.
- 사용자가 틀린 문제 수와 오답 목록을 쉽게 확인할 수 있게 한다.
- 오답 문제만 모은 문제 배열 또는 세션 입력을 생성한다.
- 복습을 시작하면 일반 세션과 구분되는 `review` 모드 흐름으로 들어가도록 준비한다.
- 오답이 하나도 없는 경우에는 복습 CTA 대신 안내 메시지를 보여준다.

이번 브랜치에서는 "오답만 다시 시작하는 흐름"까지는 포함하지만, 그 복습 결과를 누적 통계로 저장하거나 별도 히스토리로 관리하는 단계까지는 포함하지 않는다.

## State Principles

오답 복습 상태는 아래 원칙을 따라야 한다.

- 오답 여부는 직전 세션의 `QuestionResult.isCorrect`를 기준으로 판단해야 한다.
- 복습 대상 문제는 `questionId` 기준으로 식별 가능해야 한다.
- 복습 세션은 원본 세션과 구분되는 별도 흐름이어야 한다.
- 오답이 없을 때는 복습 세션이 시작되면 안 된다.
- 결과 화면 요약 상태와 복습 세션 준비 상태는 분리되어야 한다.

## Proposed Data Shape

이번 단계에서는 최소한 아래 수준의 구조가 필요할 가능성이 높다.

- `wrongQuestionIds: QuestionId[]`
- `wrongQuestions: Question[]`
- 필요 시 복습 전용 세션 구조:
  - `mode: "review"`
  - `questionIds`
  - `questionSetId`
  - 기타 최소 세션 정보

필요하다면 결과 요약 아이템에 `isCorrect` 외에 복습용 필드가 추가될 수 있다.  
핵심은 오답 추출 규칙과 복습 시작 입력을 분리하는 것이다.

## Wrong-answer Extraction Rules

오답 추출 규칙은 최소한 아래를 만족해야 한다.

- 오답 문제는 `QuestionResult.isCorrect === false`인 항목만 포함한다.
- 오답 목록 순서는 원래 세션 문제 순서를 유지하는 편이 안전하다.
- 오답 문제 목록은 결과 화면의 문제별 요약과 일치해야 한다.
- 중복된 `questionId`가 있다면 하나의 문제로 정규화해야 한다.

## Interaction Design

이번 단계에서 기대하는 상호작용은 다음과 같다.

### 1. On Result Page

- 사용자는 결과 화면에서 오답 수를 확인한다.
- 오답이 있다면 "오답 복습" CTA를 볼 수 있다.

### 2. Review Start

- 사용자가 복습 CTA를 누르면 오답 문제만 포함된 복습 흐름으로 이동한다.
- 복습 세션은 일반 모드와 구분되는 컨텍스트를 가져야 한다.

### 3. Review Session

- 복습 화면은 오답 문제만 보여준다.
- 최소한 사용자는 "현재 오답 복습 중"이라는 사실을 알아야 한다.

### 4. No Wrong Answers

- 오답이 없으면 복습 CTA 대신 축하 또는 완료 메시지를 보여준다.
- 사용자가 굳이 빈 복습 흐름으로 진입하지 않도록 해야 한다.

### 5. Non-goals in This Step

- 오답 복습 결과를 따로 저장
- 오답 복습 통계 누적
- 복습과 랜덤 모드 조합
- 복습과 시험 모드 조합

## UI Principles

- 오답 복습 CTA는 결과 화면의 다음 행동으로 자연스럽게 보이게 해야 한다.
- 오답이 없는 경우에는 사용자가 혼란스럽지 않도록 명확한 완료 상태를 보여야 한다.
- 복습 화면은 일반 퀴즈 화면과 닮되, 복습 컨텍스트가 분명해야 한다.
- 오답 목록은 숫자와 텍스트 둘 다로 이해할 수 있어야 한다.

## Review Flow Notes

이번 단계는 아래 흐름을 최소한 지원해야 한다.

1. 결과 집계에서 오답 문제 추출
2. 결과 화면에서 오답 복습 진입
3. 오답만 포함한 복습 세션 시작
4. 복습 모드 퀴즈 화면 표시

핵심은 "결과 요약"과 "복습 시작" 사이에 끊김이 없어야 한다는 점이다.

## Data Flow Notes

현재 결과 화면은 최신 세션과 활성 문제 세트를 읽고 있다.  
이번 단계에서는 아래 중 한 방향으로 정리될 가능성이 높다.

- 결과 화면에서 오답 문제 ID 목록을 계산하고 복습 진입 시 전달
- 복습 전용 세션 상태를 별도로 저장하고 `/review`가 이를 읽도록 연결

어느 방식을 택하든 이번 단계의 핵심은 "복습 화면이 오답만 안정적으로 읽을 수 있게 만드는 것"이다.

## Risks

- 결과 화면 집계와 복습 세션 시작 로직을 과하게 결합하면 유지보수가 어려워진다.
- 오답 추출 기준이 흔들리면 복습 대상이 잘못 잡힐 수 있다.
- 복습 세션이 일반 세션과 상태를 공유하면 결과 집계가 오염될 수 있다.
- 복습 화면을 별도 설계 없이 얹으면 일반 퀴즈 흐름과 역할이 모호해질 수 있다.

## Per-set Wrong Answer Revision

추가 수정에서는 오답 복습을 전역 최신 세션 기준이 아니라, 문제 세트별로 분리해서 해석할 수 있어야 한다.

- 사용자는 현재 활성 문제 세트 기준으로 해당 세트의 오답만 봐야 한다.
- 다른 문제 세트에서 틀린 문제는 현재 세트의 오답 목록에 섞이면 안 된다.
- 오답 추출 기준은 여전히 `isCorrect === false`지만, 반드시 `questionSetId` 문맥 안에서 필터링되어야 한다.
- 오답 복습 화면도 현재 세트에 해당하는 오답만 보여야 한다.
- 오답 목록 또는 복습 대상에서 특정 문제를 제거하는 기능이 있어야 한다.
- 이 삭제는 "세트별 오답 목록에서 제거"를 뜻하며, 다른 세트의 오답 기록에는 영향을 주면 안 된다.

이번 수정의 핵심은 오답 복습을 "전역 틀린 문제 모음"이 아니라, "문제 세트별 학습 루프"로 바꾸는 것이다.

### Per-set Wrong Answer Deliverables

- [x] 문제 세트별 오답 식별 규칙
- [x] 현재 활성 세트 기준 오답 목록 구성
- [x] 오답 목록/복습 대상에서 문제 제거 기능
- [x] 다른 세트 오답과 섞이지 않는 복습 흐름

### Per-set Wrong Answer Validation

- [x] 현재 세트의 오답만 결과 화면과 복습 화면에 나타나야 한다.
- [x] 다른 세트 오답이 현재 세트 복습에 섞이면 안 된다.
- [x] 특정 오답 삭제가 다른 세트 오답에는 영향을 주면 안 된다.
- [x] 오답 삭제 후 현재 세트 복습 목록이 즉시 갱신되어야 한다.

## Review Retry Revision

추가 수정에서는 오답 복습도 일반 학습처럼 이전 문제로 돌아갈 수 있어야 하고, 복습 회차 표기도 실제 재도전 횟수에 맞게 증가해야 한다.

- 오답 복습에서 이미 푼 이전 문제로 돌아갈 수 있어야 한다.
- 이전 문제로 돌아왔을 때 기존 정답/오답 상태가 유지되어야 한다.
- 남은 오답으로 다시 시작할 때 회차 표기는 항상 `2차`로 고정되면 안 된다.
- 2차 이후에는 3차, 4차처럼 실제 재시도 횟수가 자연스럽게 올라가야 한다.

### Review Retry Deliverables

- [x] 오답 복습 이전 문제 이동
- [x] 복습 중 재방문 문제 상태 복원
- [x] 회차 기반 완료/재시작 문구

### Review Retry Validation

- [x] 오답 복습에서 이전 문제 이동이 가능해야 한다.
- [x] 이전 문제로 돌아왔을 때 이미 제출한 결과가 유지되어야 한다.
- [x] 2차 이후 재도전은 3차, 4차로 증가해야 한다.
