# feature/random-quiz-mode Plan

## Goal

활성 문제 세트를 그대로 사용하되, 문제 순서를 무작위로 섞어서 풀이할 수 있는 랜덤 모드를 구현한다.

## Branch Intent

이 브랜치는 "정해진 순서로 푸는 세션"에 "무작위 순서 세션"을 추가하는 단계다.

- [x] 랜덤 모드에서만 적용되는 셔플 순서를 만든다.
- [x] 랜덤 모드임을 문제 화면과 결과 화면에서 분명히 표시한다.
- [x] 셔플된 순서에서도 결과 집계와 오답 복습이 일관되게 동작하도록 구조를 유지한다.
- [x] 다음 단계인 시험 모드와 충돌하지 않도록 랜덤 모드의 책임을 분명히 유지한다.

이번 단계의 핵심은 "문제 순서를 바꾸는 것"이지, 채점 방식이나 결과 집계 규칙을 새로 만드는 것이 아니다.

## Scope

- 랜덤 모드 진입 시 셔플 순서 생성
- 셔플된 순서대로 문제 렌더링
- 랜덤 모드 라벨 표시
- 랜덤 세션의 결과 저장
- 랜덤 세션 결과 화면 일관성 유지
- 랜덤 세션 이후 오답 복습 기준 유지

## Detailed Scope

이번 단계에서 다뤄야 하는 핵심은 아래와 같다.

- 현재 `questions` 배열 기반 세션에 랜덤 순서를 적용할 수 있는 구조를 만든다.
- 랜덤 모드일 때만 문제 순서를 섞고, 일반 모드에는 영향을 주지 않게 한다.
- 셔플된 순서를 세션 전체에서 고정해 같은 문제를 중간에 다시 섞지 않도록 한다.
- 결과 저장 시에도 실제 풀이 순서가 반영되도록 `questionIds`와 결과 배열을 정리한다.
- 결과 화면과 오답 복습이 셔플 여부와 무관하게 정확하게 동작하도록 보장한다.

이번 브랜치에서는 무작위 문제 선택이나 일부 문제만 추출하는 기능까지는 포함하지 않는다. 전체 문제를 한 번씩 모두 사용하는 것이 기본이다.

## State Principles

랜덤 모드 상태는 아래 원칙을 따라야 한다.

- 셔플은 세션 시작 시 한 번만 생성되어야 한다.
- 같은 세션 안에서는 셔플 순서가 고정되어야 한다.
- 랜덤 모드 여부는 `QuizMode` 또는 동등한 세션 정보로 식별 가능해야 한다.
- 결과 집계는 셔플 여부와 무관하게 동일한 규칙으로 계산되어야 한다.
- 오답 추출은 셔플 순서가 아니라 `questionId` 기준으로 동작해야 한다.

## Proposed Data Shape

이번 단계에서는 최소한 아래 수준의 구조가 필요할 가능성이 높다.

- `mode: "random"`
- 셔플된 `questionIds`
- 필요 시 `shuffledQuestions: Question[]`

핵심은 화면 표시 순서와 결과 식별 기준을 분리하는 것이다.  
결과 식별은 여전히 `questionId` 기준으로 가야 하고, 셔플은 "보여주는 순서"에만 영향을 줘야 한다.

## Shuffle Rules

랜덤 셔플 규칙은 최소한 아래를 만족해야 한다.

- 전체 문제를 중복 없이 한 번씩 포함해야 한다.
- 문제 수가 같아야 하며 누락이 없어야 한다.
- 랜덤 모드 세션 안에서는 순서가 다시 바뀌면 안 된다.
- 결과 화면과 복습 화면은 셔플된 풀이 순서를 반영하되, 식별은 `questionId` 기준으로 유지해야 한다.

## Interaction Design

이번 단계에서 기대하는 상호작용은 다음과 같다.

### 1. From Home

- 사용자는 홈에서 랜덤 모드를 선택한다.
- 랜덤 모드로 진입하면 문제 순서가 무작위로 정해진다.

### 2. During Quiz

- 문제 화면 헤더에서 현재 세션이 랜덤 모드임을 알 수 있어야 한다.
- 진행 흐름 자체는 일반 모드와 최대한 비슷해야 한다.

### 3. Session Complete

- 마지막 문제를 끝내면 결과 화면으로 이동한다.
- 결과 화면은 랜덤 모드 세션 결과라는 점을 잃지 않으면서도 집계 규칙은 동일해야 한다.

### 4. Review After Random Session

- 오답 복습은 랜덤 세션 결과에서도 정상 동작해야 한다.
- 복습 대상 문제는 셔플 여부와 상관없이 정확히 추출되어야 한다.

### 5. Non-goals in This Step

- 일부 문제만 랜덤 선택
- 난이도 기반 랜덤
- 시험 모드와 랜덤 조합
- 랜덤 시드 고정/공유 기능

## UI Principles

- 랜덤 모드라는 사실이 분명해야 하지만, 화면 구조는 기존 퀴즈 흐름과 크게 달라지지 않는 편이 좋다.
- 사용자는 셔플이 한 번 적용된 뒤 세션이 안정적으로 진행된다고 느껴야 한다.
- 결과 화면과 복습 흐름은 랜덤이라는 이유로 더 복잡해 보이면 안 된다.

## Data Flow Notes

현재 퀴즈 세션은 `questionIds`와 `results`를 저장하고 있다.  
이번 단계에서는 아래 흐름이 중요하다.

- 랜덤 모드 진입 시 셔플된 문제 순서 생성
- 그 순서를 기준으로 세션 진행
- 완료 시 셔플된 `questionIds`를 결과 세션에 저장
- 결과 요약과 오답 복습은 저장된 `questionIds`와 `results`를 기준으로 계산

즉, 랜덤 모드의 핵심은 "세션 시작 시점의 순서 생성"이지, 결과 화면에서 다시 셔플을 해석하는 것이 아니다.

## Risks

- 셔플을 렌더링 중에 다시 계산하면 세션 안정성이 깨질 수 있다.
- 결과 저장 시 실제 풀이 순서와 `questionIds`가 어긋나면 결과 화면이 잘못 표시될 수 있다.
- 랜덤 모드와 일반 모드 로직을 과하게 분기하면 유지보수가 어려워질 수 있다.
- 오답 복습이 셔플 순서에 종속되면 잘못된 문제를 복습하게 될 수 있다.

## Notes

- 셔플은 세션 시작 시점에만 일어나야 하며, 이후 흐름은 일반 세션과 최대한 닮게 두는 편이 안전하다.
- 결과와 복습의 신뢰도를 깨지 않는 랜덤 모드가 가장 중요하다.
