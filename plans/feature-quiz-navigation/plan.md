# feature/quiz-navigation Plan

## Goal

현재 문제를 제출하고 해설을 확인한 뒤, 다음 문제로 자연스럽게 이동하고 마지막 문제에서는 세션 종료 흐름으로 넘어갈 수 있는 내비게이션 구조를 만든다.

## Branch Intent

이 브랜치는 "한 문제를 푸는 화면"을 "여러 문제를 순서대로 진행하는 세션 화면"으로 확장하는 단계다.

- [x] 현재 문제 인덱스를 기준으로 문제를 순차적으로 렌더링한다.
- [x] 제출과 해설 확인 이후에만 다음 문제로 이동할 수 있게 한다.
- [x] 마지막 문제에서는 다음 문제 이동 대신 세션 종료 흐름을 제공한다.
- [x] 다음 단계인 결과 화면 구현이 자연스럽게 올라갈 수 있도록 세션 종료 시점의 상태를 정리한다.

이번 단계의 핵심은 "이동"이지 "집계"가 아니다.  
즉, 결과를 계산하는 상세 화면은 아직 만들지 않지만, 결과 화면으로 넘어갈 수 있는 흐름은 준비해야 한다.

## Scope

- 현재 문제 인덱스 상태 도입
- 현재 문제를 인덱스 기준으로 렌더링
- 다음 문제 이동 버튼
- 제출 이후에만 이동 가능하도록 제어
- 마지막 문제에서 종료 버튼 또는 결과 보기 버튼 제공
- 결과 화면 전이를 위한 최소 라우팅 연결
- 문제 전환 시 선택/제출/해설 상태 초기화

## Detailed Scope

이번 단계에서 다뤄야 하는 핵심은 아래와 같다.

- 지금까지 첫 문제 고정으로 렌더링하던 구조를 `currentQuestionIndex` 기반 구조로 변경
- `currentQuestion`를 `questions[currentQuestionIndex]`에서 가져오도록 수정
- 사용자가 제출과 해설 단계를 거친 뒤 다음 문제로 넘어가게 제어
- 다음 문제로 이동하면 이전 문제의 선택/제출/해설 상태가 현재 문제에 남지 않도록 초기화
- 마지막 문제에서는 "다음 문제"가 아니라 "결과 보기" 또는 동등한 종료 CTA를 제공
- 결과 화면 구현 전이라도 `/result`로 이동 가능한 최소 연결을 준비

이번 브랜치의 핵심은 "문항 간 이동 규칙을 확립하는 것"이며, 세션 전체의 정답률 계산까지 책임지지는 않는다.

## State Principles

세션 이동 상태는 아래 원칙을 따라야 한다.

- 현재 문제는 항상 하나의 인덱스로 표현되어야 한다.
- 인덱스는 문제 수 범위를 벗어나면 안 된다.
- 문제 전환 시 문제 단위 UI 상태는 초기화되어야 한다.
- 이동 가능 여부는 제출 상태에 종속되어야 한다.
- 마지막 문제 판정은 현재 인덱스와 전체 문제 수에 기반해 단순하게 계산되어야 한다.

## Proposed State Shape

이번 단계에서는 최소한 아래 수준의 상태 구조가 적절하다.

- `currentQuestionIndex: number`
- `currentQuestion: Question | undefined`
- `isSubmitted: boolean`
- `isCorrect: boolean | null`
- `isExplanationOpen: boolean`

그리고 유도 값으로 아래 정보가 필요할 가능성이 높다.

- `isLastQuestion: boolean`
- `currentQuestionNumber: number`
- `canGoToNextQuestion: boolean`

핵심은 문제 이동 상태와 문제 풀이 상태를 분리하되, 이동 가능 조건은 풀이 상태에 의존하도록 두는 것이다.

## Interaction Design

이번 단계에서 기대하는 상호작용은 다음과 같다.

### 1. Before Submit

- 사용자는 현재 문제를 읽고 답을 선택한다.
- 아직 다음 문제로 갈 수 없다.
- 이동 CTA가 보이더라도 비활성 상태이거나 숨겨져야 한다.

### 2. After Submit

- 사용자는 정답/오답과 해설을 확인한다.
- 다음 문제로 넘어갈 수 있는 CTA가 활성화된다.

### 3. Move To Next Question

- 사용자가 다음 문제 CTA를 누르면 다음 인덱스로 이동한다.
- 새 문제에서는 선택, 제출, 해설 상태가 모두 초기화된다.
- 진행 표시도 새 문제 번호에 맞게 갱신된다.

### 4. Last Question

- 마지막 문제에서는 다음 문제 CTA 대신 세션 종료 CTA가 나타난다.
- 이 CTA는 결과 화면 또는 종료 흐름으로 연결된다.

### 5. Non-goals in This Step

- 누적 정답 수 계산
- 문제별 결과 목록 작성
- 오답만 추리기
- 통계 저장

## UI Principles

- 이동 CTA는 제출/해설 흐름을 방해하지 않게 배치해야 한다.
- 사용자는 "지금 무엇을 할 수 있는지"를 명확히 이해할 수 있어야 한다.
- 마지막 문제에서는 CTA 문구가 분명히 달라져야 한다.
- 문제 전환 시 이전 문제의 피드백이 남아 있으면 안 된다.
- 진행 표시와 내비게이션은 서로 같은 현재 인덱스를 바라봐야 한다.

## Navigation Rules

세션 이동 규칙은 최소한 아래를 만족해야 한다.

- 현재 문제가 제출되기 전에는 다음 문제로 이동할 수 없다.
- 현재 문제가 마지막 문제가 아니면 다음 문제로 이동할 수 있다.
- 현재 문제가 마지막 문제이면 종료 흐름으로 전환해야 한다.
- 인덱스 증가 로직은 범위를 벗어나지 않도록 보호해야 한다.

## Resume Navigation Revision

이후 리비전에서는 "문제풀이 중 홈으로 나갔다가 다시 이어서 풀기" 흐름을 추가로 지원해야 한다.

- 퀴즈 화면에는 세션을 강제로 종료하지 않고 홈으로 나갈 수 있는 CTA가 있어야 한다.
- 홈으로 이동할 때 현재 세션은 "완료된 세션"이 아니라 "진행 중 세션"으로 저장되어야 한다.
- 다시 `/quiz`로 진입하면 저장된 진행 중 세션이 있을 경우 그 시점으로 복귀할 수 있어야 한다.
- 복귀 기준은 최소한 아래를 포함해야 한다.
  - `mode`
  - `questionSetId`
  - `questionIds`
  - `currentQuestionIndex`
  - 이미 제출된 결과 목록
- 복귀 시에는 아직 제출하지 않은 임시 선택 상태까지 복원할지 여부를 정책으로 명확히 정해야 한다.

### Recommended Resume Policy

현재 구조에서는 아래 정책이 가장 안전하다.

- 이미 제출한 문제 결과는 복원한다.
- 현재 문제 인덱스는 복원한다.
- 문제 순서는 복원한다.
- 아직 제출하지 않은 선택 상태(`selectedChoiceIndex`)는 복원하지 않는다.
- 해설 열림 상태도 복원하지 않고 현재 문제 진입 시 닫힌 상태로 둔다.

이 정책을 따르면 저장 구조가 단순해지고, "부분적으로 선택만 해 둔 상태" 때문에 잘못된 UI가 복원되는 위험을 줄일 수 있다.

### Resume CTA Rules

- 퀴즈 화면에는 `홈으로 이동` 또는 동등한 CTA가 있어야 한다.
- 이 CTA는 현재 세션을 버리지 않고 저장한 뒤 홈으로 이동해야 한다.
- 홈 화면에서는 진행 중 세션이 있을 때 "이어 풀기" 진입을 제공할 수 있어야 한다.
- 저장된 진행 중 세션이 현재 활성 문제 세트와 맞지 않으면 복귀 대신 새 세션 시작 또는 안전한 초기화로 전환해야 한다.

### Resume Non-goals

- 세션 중간 이탈 시 결과 화면 집계
- 여러 개의 진행 중 세션 동시 보관
- 브라우저 간 동기화
- 미제출 선택 상태의 완전 복구

## Free Navigation Revision

이번 수정에서는 일반 문제풀이 모드에서만 "이전 문제 / 다음 문제"를 자유롭게 오갈 수 있는 탐색 기능을 추가해야 한다.  
다만 시험 모드는 연속 응시 흐름을 유지해야 하므로, 이 자유 이동 규칙을 적용하면 안 된다.

- 일반 모드에서는 현재 문제를 기준으로 이전 문제와 다음 문제로 이동할 수 있어야 한다.
- 랜덤 모드도 학습 성격이므로 일반 모드와 같은 자유 이동 규칙을 재사용할지 여부를 명확히 정해야 한다.
- 시험 모드에서는 기존처럼 현재 문항 제출 후에만 다음 문항으로 진행하고, 이전 문항으로 돌아가면 안 된다.
- 자유 이동은 "문제 순서를 바꾸는 것"이 아니라, 이미 정해진 세션 문제 순서 안에서 현재 인덱스를 옮기는 기능이어야 한다.
- 문제를 아직 제출하지 않은 상태에서 다른 문제로 이동하는 경우, 미제출 선택 상태를 저장하거나 복원하지 않는 현재 정책과 충돌하지 않도록 해야 한다.

### Recommended Free Navigation Policy

현재 구조에서는 아래 정책이 가장 안전하다.

- 일반 모드와 랜덤 모드에서만 이전/다음 이동 CTA를 보여준다.
- 시험 모드에서는 기존 CTA만 유지하고 이전 이동 기능은 제공하지 않는다.
- 이미 제출한 문제는 다시 들어왔을 때 저장된 결과 상태로 보인다.
- 아직 제출하지 않은 문제에서 다른 문제로 이동하면, 미제출 선택 상태는 유지하지 않는다.
- 자유 이동은 결과 집계를 새로 만들지 않고, 기존 `questionResults`를 그대로 재사용한다.

### Free Navigation Rules

- 첫 문제에서는 `이전 문제` CTA를 비활성화하거나 숨겨야 한다.
- 마지막 문제가 아니면 `다음 문제` CTA로 이동 가능해야 한다.
- 마지막 문제에서도 학습 모드에서는 결과 화면 전이 CTA가 유지되어야 한다.
- 제출 여부와 무관하게 이동은 가능하지만, 제출된 문제와 미제출 문제의 상태 표현은 명확히 구분되어야 한다.
- 이동 후에는 현재 문제에 저장된 결과가 있으면 그 결과를 복원하고, 없으면 새 문제처럼 보여야 한다.

### Free Navigation Non-goals

- 시험 모드에서 이전 문항 이동 허용
- 문제 번호 목록 전체 네비게이터 구현
- 북마크형 점프 네비게이션
- 미제출 임시 답안 자동 저장

## Result Transition Notes

이번 단계는 결과 화면 상세 구현 전 단계이지만, 전이 지점은 준비해야 한다.

- 마지막 문제 종료 CTA는 `/result`로 이동할 수 있어야 한다.
- 아직 결과 집계 데이터가 없더라도, 최소한 라우팅 구조는 끊기지 않아야 한다.
- 다음 단계 `feature/result-summary`에서 이 전이 지점 위에 집계와 요약 UI가 올라갈 수 있어야 한다.

## Data Usage Notes

이번 단계는 활성 문제 세트 전체 문제 배열을 실제로 순회하기 시작하는 첫 단계다.

- `questions[currentQuestionIndex]`를 현재 문제로 사용
- `questions.length`를 진행 수치와 마지막 문제 판정에 사용
- 활성 문제 세트가 없는 경우에는 기존 빈 상태를 그대로 유지

이 단계부터는 더 이상 "첫 문제 하나만 보여주는 화면"이 아니라 "문제 배열을 순서대로 진행하는 세션 화면"이 된다.

## Out of Scope

- 결과 집계 상세 UI
- 정답 수/오답 수 계산 결과 표시
- 오답 문제 목록 구성
- 랜덤 모드 특화 순서 처리
- 시험 모드 특화 흐름
- 다중 진행 세션 보관

- [x] 홈 이동 후 다시 `/quiz`로 들어오면 직전 진행 문제 번호로 복귀해야 한다.
- [x] 이미 제출한 문제 결과는 유지되어야 한다.
- [x] 아직 제출하지 않은 선택 상태는 의도한 정책대로 초기화되거나 복원되어야 한다.
- [x] 문제 세트가 바뀐 경우 오래된 진행 세션을 잘못 복원하면 안 된다.
- [x] 완료된 세션과 진행 중 세션이 서로 혼동되지 않아야 한다.

### Free Navigation Validation

- [x] 일반 문제풀이 모드에서 이전 문제로 이동할 수 있어야 한다.
- [x] 일반 문제풀이 모드에서 제출 전에도 다음 문제로 이동할 수 있어야 한다.
- [x] 이미 제출한 문제로 돌아가면 기존 결과 상태가 다시 보여야 한다.
- [x] 미제출 문제로 이동하면 저장된 결과가 없는 한 새 문제 상태로 보여야 한다.
- [x] 첫 문제에서는 이전 이동이 잘못 활성화되면 안 된다.
- [x] 시험 모드에서는 이전 이동 CTA가 나타나면 안 된다.

## Success Criteria

아래 조건을 만족하면 `feature/quiz-navigation` 단계가 완료된 것으로 본다.

- [x] 사용자가 여러 문제를 순서대로 진행할 수 있다.
- [x] 제출 후에만 다음 문제로 넘어갈 수 있다.
- [x] 마지막 문제에서는 세션 종료 흐름으로 전환된다.
- [x] 다음 단계에서 결과 집계와 요약만 얹으면 되도록 구조가 준비된다.

## Risks

이번 단계에서 특히 주의해야 할 위험은 다음과 같다.

- 인덱스 상태와 선택/제출/해설 상태가 어긋나면 잘못된 문제에 결과가 붙을 수 있다.
- 마지막 문제 판정이 불안정하면 결과 화면 진입이 깨질 수 있다.
- 문제 전환 시 상태 초기화가 누락되면 이전 문제의 정답/오답 UI가 다음 문제에 남을 수 있다.
- 결과 집계까지 한 번에 넣으려 하면 다음 브랜치 경계가 무너진다.

## Notes

- 이번 브랜치의 핵심은 "한 문제 화면"을 "문제 세션 화면"으로 바꾸는 것이다.
- 세션 이동 규칙이 안정되어야 이후 결과, 랜덤, 시험 모드가 자연스럽게 이어진다.
- 이 단계에서는 집계보다 이동 규칙의 명확성이 더 중요하다.

## Completion Status

완료. 현재 문제 인덱스 기반 렌더링, 제출 후 다음 문제 이동, 마지막 문제 종료 CTA, 결과 라우트 연결, 타입 점검까지 반영했다.

리비전도 완료. 퀴즈 화면에서 홈으로 저장 후 이동할 수 있는 CTA를 추가했고, 진행 중 세션을 별도 저장소에 기록한 뒤 `/quiz` 재진입 시 현재 문제 인덱스와 제출 결과를 복원하도록 연결했다. 미제출 선택 상태와 해설 열림 상태는 복원하지 않는 정책도 반영했다.

자유 이동 수정도 완료. 일반 문제풀이 모드에서는 이전/다음 문제 탐색을 허용했고, 이미 제출한 문제 결과는 그대로 복원되며 미제출 문제는 새 상태로 유지된다. 시험 모드는 기존의 제한된 이동 규칙을 그대로 유지한다.

## Session Reset Revision

이후 수정에서는 사용자가 현재 퀴즈 세션을 버리고 처음부터 다시 시작할 수 있는 초기화 기능도 필요하다.

- 퀴즈 화면 안에서 `처음부터 다시 시작` 같은 명시적인 CTA가 있어야 한다.
- 이 동작은 현재 문제 인덱스와 결과 배열을 초기화하고 첫 문제부터 다시 시작해야 한다.
- 진행 중 세션이 저장돼 있었다면 함께 정리되어야 한다.
- 홈 화면에서 `restart=1` 같은 명시적 진입을 사용할 경우에도 이어풀기 복귀보다 초기화가 우선해야 한다.

### Session Reset Deliverables

- [x] 퀴즈 화면의 처음부터 다시 시작 CTA
- [x] 현재 세션 인덱스/결과 초기화
- [x] 진행 중 세션 정리 후 새 세션 시작
- [x] 명시적 재시작 진입 시 이어풀기보다 초기화 우선

### Session Reset Validation

- [x] 퀴즈 화면에서 처음부터 다시 시작할 수 있어야 한다.
- [x] 재시작 후 첫 문제부터 새 상태로 보여야 한다.
- [x] 기존 진행 중 세션은 함께 정리되어야 한다.
- [x] 홈에서 재시작 진입 시 자동 이어풀기보다 새 세션 시작이 우선해야 한다.

## Per-set Resume Navigation Revision

추가 수정에서는 퀴즈 재진입 복귀 규칙도 "전역 진행 중 세션 1개"가 아니라, 현재 문제 세트 기준의 이어풀기 세션을 찾는 방식으로 바뀌어야 한다.

- 퀴즈 화면은 현재 활성 세트와 범위, 모드에 맞는 진행 중 세션만 복원 대상으로 봐야 한다.
- 사용자가 다른 세트로 문제를 풀고 있었다 해도, 현재 세트에 맞는 세션이 있으면 그것을 우선 복원해야 한다.
- 반대로 현재 세트와 맞지 않는 세션이 있다고 해서 이를 정리하거나 덮어쓰면 안 된다.
- 즉 퀴즈 화면의 복원 로직은 "지금 들어온 세트/범위/모드"에 해당하는 세션을 찾고, 없으면 새 세션을 시작하는 방식이 적절하다.
- 홈으로 이동할 때도 현재 세트의 진행 세션만 갱신해야 하며, 다른 세트의 이어풀기 세션을 건드리면 안 된다.

### Per-set Resume Navigation Deliverables

- [x] 현재 세트/범위/모드 기준 세션 복원 규칙
- [x] 다른 세트 세션을 보존하는 홈 이동 저장 규칙
- [x] 현재 세트에 매칭되는 세션이 없을 때만 새 세션을 시작하는 흐름

### Per-set Resume Navigation Validation

- [x] 다른 세트 세션이 존재해도 현재 세트 세션 복원에는 영향이 없어야 한다.
- [x] 현재 세트와 맞지 않는 세션 때문에 잘못 복원되면 안 된다.
- [x] 홈 이동 후 현재 세트 세션만 갱신되고 다른 세트 세션은 유지되어야 한다.
